Wrapping up Week 1

What is the difference between margin, border, and padding?

Margins are to help push away content. Borders are used to help encircle the content. Padding helps to push in and cushion your content.

What are the best practices associated with using classes vs. ids?

It's best to use classes over ids. That way you can use the same class on multiple elements OR multiple classes on the same element! When you use ids, you can only use that ID on one element once on a page.

To complicate it more, browsers are more finicky on how ids are used, but CSS doesn't care. In CSS there is nothing different between the two. Therefore it's best to proceed with caution when implementing ids. Ask yourself, could this styling be used throughout more places on your site?

What are the differences between relative, absolute, fixed, and static positioning?

Static Position is the default type of positioning for an element. Elements stack one after another in how they were coded in HTML.

Relative Position will move on it's own and can overlap over other elements using z-index and top, bottom, left, right directions. If you specify top: 100px; and left: 50px; it will away from these locations by the specified pixels. What happens is that where it originally was will continue to hold as a placeholder and will not affect the flow of static elements around it.

Fixed Position is positioned based on the browser window and where you direct it to go with top, right, bottom, and left. A fixed position element will stay in that place even when the page is scrolled.

What does it mean to display inline vs using inline block?

Display inline means to allow the elements to flow together and appear continuous. Display inline block means that they will block by having clear defined boundaries and will sit side by side.
